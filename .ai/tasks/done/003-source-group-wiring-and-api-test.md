## Context

**Resolver root pattern:** `internal/backend/resolvers/resolver.go` holds a single `Resolver`
struct with one field per resource service. Add `SourceGroupSvc *services.SourceGroupService`
following the same pattern as `DestinationGroupSvc`.

**Resolver implementation pattern:** `internal/backend/resolvers/destination_group.resolvers.go`.
Each method body is a one-liner delegating to the service and returning its result directly.
Example:
```go
func (r *mutationResolver) CreateSourceGroup(ctx context.Context, sourceGroup model.SourceGroupInput) (*model.SourceGroup, error) {
    return r.SourceGroupSvc.CreateSourceGroup(ctx, sourceGroup)
}
```

**AppContext pattern:** `internal/backend/appcontext/context.go`. The `AppContext` struct has
one field per service; `NewAppContext` constructs each service with the store. Follow the
same pattern as `DestinationGroupSvc`:
```go
SourceGroupSvc: services.NewSourceGroupService(s),
```

**Router pattern:** `internal/backend/handlers/graphql/router.go` constructs a `&resolvers.Resolver{...}`
passing all service fields. Add `SourceGroupSvc: appCtx.SourceGroupSvc`.

**API test file pattern:** Follow any existing file in `api-tests/` (e.g. `CarrierGraphQL.http`
or `DestinationGroupGraphQL.http` if present). The file must cover:
1. `sourceGroupList` — default page (no filter)
2. `sourceGroupList` — with wildcard filter on `groupName`
3. `sourceGroupList` — with filter on `region`
4. `countSourceGroup` — no filter
5. `countSourceGroup` — with filter
6. `sourceGroupByGroupName` — existing record (use seed data, e.g. `HOME`)
7. `createSourceGroup` — new record
8. `updateSourceGroup` — update an existing record
9. `deleteSourceGroup` — delete by group name

Use realistic sample data consistent with `db/migrations/000005_source_groups.up.sql`:
existing seed entries include `WORLD`, `EUROPE`, `N-AMERICA`, `MIDDLE-EAST`, `AFRICA`,
`S-AMERICA`, `OCEANIA`, `NATIONAL`, `HOME`.

The HTTP file must use `{{baseUrl}}` and `{{bearerToken}}` template variables exactly as
other files in `api-tests/` do.

**NOTE:** The resolver stub file (`source_group.resolvers.go`) was generated by gqlgen in
Task 002. Do NOT regenerate gqlgen in this task — only fill in the method bodies. Regenerating
gqlgen would require the schema to already be in place (it is) but could also overwrite the
edits you make to the stub file. If the stub file needs corrections, edit it directly.
